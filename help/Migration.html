Migration

 

Note: Ensure you are authenticated as per process described above before executing this step.

Migration Concepts:

 

Name for Migration

Migration is core offering for this software. Each time when migration is done a unique Migration Id is generated. Hence in order to identify a Migration you can give it a name.

Entering Source and Destination mapping

Application will require a  mapping file in CSV format to be uploaded.  Mapping file should contain in one row Source User ID in first column and Destination account information in Second Column.

Upload a comma separated file with Source and destination mapping. Source user or group (email) in first column, equivalent destination user or group or site URL to be represented in second column. No title or heading row required. Source names do not accepts URL (just User or Group IDs). If using Site URL for destination, site should be already created in destination tenant. File size less than 14 MB is accepted

Multiple Source and Destination User IDs can be added as additional row in CSV file that need a migration. 

If source and destination tenants are authenticated using same User IDs you can simply enter comma separated list of user Ids requiring migration, or upload a simple comma separated csv file for user ids

Entering Destination folder

It is recommended you create a new folder at destination tenant for doing the migration. To do this enable checkbox for "Check if source destination is not root folder" and enter folder name. Folders are automatically created if not present during migration. 

 

File conflict behavior

Note when encountering a same folder at destination when doing migration system does not creates a new folder at destination but only copies files under that folder from Source to Destination. However when encountering a file with same name behavior such as Skip, Replace or Rename can be selected.

Usages of such behavior is explained below:

When doing large migration or users having lot of file contents follow the below steps:

Do a first pass Migration. For First Pass Migration option also select rename option when encountering files of same name. It is impossible that this scenario will occur because files will be created first time in first pass migration, however in cases when two file exists of same name (specially in case of Google) you will be able to migrate the other file as well with a number appended to it. You are allowed not to select Rename in first pass migration but in that case if you see reports files having error with “Conflict”  for first pass migration, they are probably files with same name.

Other options are used when you are trying to overwrite the contents of files you have already migrated by rewrite options or just skipping/ignoring files if same file names are encountered. Normally they are applied when you are doing a second pass migration. For files that have encountered failure and after respective correction is done you can select this option to ignore files with same name or rewrite them.

During second pass migration application currently do not checks or compare timestamps at source and destination folder. 

During Migration 

Migrator allows only one Migration request at a time. Hence if a Migration is in progress no other users are allowed to submit new migration request.

Once Migration starts users are shown key statistics from Migration in Migration Dashboard that appears below the submit button. Migration Dashboard is self-explanatory.

Key Notes:

  • Once User starts migration, they can close the browser if needed. If they came back to migration page they can see the migration status.
  • User can wish to stop the migration. Option to stop the migration appears after migration has started.
  • If Migration is finished, detailed reports of migration can be retrieved from the reports tab.
