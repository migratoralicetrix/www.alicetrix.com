Syncing Email migration 

 

Keep this checkbox on as default unless target/destination tenant has no mails in the mailbox. If destination is empty, this option can be unchecked to enable a faster first pass migration. 

This option is used for Sync Migration to ensure mail items that are already migrated are not duplicated in Source Tenant. 

System creates a local database for all migrated emails to check emails in that database first before attempting to write email in destination mailbox. System also checks source tenant if email was not found in database before migrating mail item if this option is checked. 

 

This checkbox can be turned on if actual migration is not intended. In this option System just mimics the migration process and generates reports on all mail items and mail folder that needs to be migrated. Give a suitable name to this migration to ensure they can be separately reported or pulled out from reporting tab. 

Hence in this option, no emails are migrated from source account, source mailbox details are reported, destination is just checked for proper authorization, and availability of requisite accounts and folders in mailbox. Use Reports tab for detailed list of items to be migrated.
