Calendar Forwarding

This option appears for few supported source tenants (Ex. Google)  Calendars in user's mailbox are forwarded to destination user email Id. 
This is NOT a calendar copy function, calendars instance that appear in future for migrated users, are just forwarded to destination user,  
This is a Beta feature. Please reach out to us if you have a source mailbox from where Calendar needs to be moved/forwarded, we may have customized solution available for you at an additional cost/services. 

Below is folder behaviour option:
Typically deleted folder from known sources (Exchange or Google) are not migrated. Below option are available to enable few of folders that typically do not need migration (This options only work for known sources Exchange or Google). 
Check this box to not to migrate Draft folder. This is checked by default. 
Check this box to not to migrate Spam/Junk emails. 
Enter comma separated folder name for folders in mailbox you do not wish to migrate. Leave blank if no folder requires skipping. This option is particularly useful in migration IMAP sources, as system may not know if folders are not required to be migrated. 

GMAIL Mail Migration behavior

Gmail stores its mail data with a tag or label. A user can use multiple lables for same email. Lables in Gmail are converted to folder in destination tenant. Hence, same email can appear multiple times if they are tagged to multiple labels. This behavior is currently found to be inconsistent. System will try not to migrate same email again if "Check if this is Sync migration" checkbox is checked.
