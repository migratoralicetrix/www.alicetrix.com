Introduction

Alicetrix tries to bring you web application-based File Migrator for content management providers. Web based application ensure your app is server based and able to handle millions of transactions at a time.

Our help section is divided into below areas:

  • Setup
  • Remediate
  • Migrate
  • Report

Below terminology we will use frequently in help file and application.

  • Source - Tenant from where User files/folder will be migrated
  • Destination - Tenant where User files and folder will be migrated

Core differentiators of Alicetrix File Migrator are:

  • Multi-Threaded Environment – while a single web application of File Migrator only migrates one user at a time (multiple users migration can be requested in same migration request), files of single users are uploaded read and uploaded into destination tenant exactly at the same time. Also larger files are broken into small pieces for upload, and for each small segment of files multithreading is done. It is particularly beneficial in migrating files of very large size and approximately 20 times faster than nearest file migration software in market. Further multi-threading can also be achieved, discussed in Migration section to stand up more instances of Web Application. Multiple instances of Web Application that can be initiated on a click of button in Cloud environment can migrate TBs of data on a single day.
  • Starts Migrating at once and, does not create a repository of files and folders before migration. It builds repository while doing migration. This is particularly beneficial in case of Users/Sites having very large data and lot of files in hundred thousands or millions.
  • Multiple-Migration Options: Multiple Migration Options are available for migrations that is first-time migration, resyncing of migration (in case user files are not updated, very fast resyncing allowing for migration for files which were newly created or left out due to some failures), folder to folder migration for a specific user etc. are available.
  • Permission Migration – Ability to migrate permission using Various algorithms are available.
  • Managed failovers - Complex algorithm built to ensure failures are handled in most case automatically by the application through retrials, wait, requeuing of file transfers or auto-correction in certain cases.
  • Security with Microsoft Azure Infrastructure – Migrator relies on secured Microsoft Azure Infrastructure for migration and additionally it is not a SaaS product but individual instance of this application are deployed through Azure Marketplace into customer’s tenant’s/environment. This ensures advanced security and data encapsulation which is sometimes needed for highly secure enterprise.
  • Remediation: Unlike few other migration tools, Alicetrix File Migrator allows to generate reports on Source Tenant. It also corrects file names if users opt for that option, getting rid of special characters etc. (for OneDrive Destination Tenant)
  • Data Storage: Migrator relies on local database for storage of files transaction information in Web App Server itself. Local database storage ensures roundtrip in accessing database is minimized ensuring faster migration. Auto caching is enabled ensuring Database access is only done a specific period of times.
  • Auto-conversions for Google: Migrates Google Docs as well, Sheets, Images into respective file outputs.
  • No File Data storage in Web App: Data read is never stored on drives.
  • Complete Reports: Complete reports are made available using excel exports.
  • Unique Dashboard in Web App: Windows client like online updates on status of migration is available and mobile friendly interface.
  • No stickiness to User Sessions: Once user Migration request is initiated, users can close the browser if needed. When they come back system displays the current migration status.
  • Admin Controls: Allows for granular admin controls on few aspects like retrials when failure, wait times etc. for more flexibility of setting up optimized speed of migration in certain cases.
